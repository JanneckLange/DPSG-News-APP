## 2. Scope of this Policy

This Policy describes how personal data is processed when you:

- install and use the app,
- receive push notifications,
- use feedback and analytics features,
- interact with API endpoints and related infrastructure.

## 3. Data We Process

Depending on your usage and device settings, we may process the following categories:

- Technical connection data (for example IP address, timestamps, request metadata).
- Device and app context data (for example operating system, app version, language).
- In-app usage events (for analytics purposes if enabled).
- Push-related technical identifiers (for example APNS/FCM token handling by platform services).
- Data you actively submit in the app (for example event content entered in author mode).

## 4. How DPSG News currently works

Based on the current implementation:

- The app does not require a user account for normal usage.
- Push notifications are handled through Firebase Cloud Messaging topics.
- Wiredash can be enabled for feedback and analytics events.
- API calls are sent to the configured backend URL.

## 5. Purposes and Legal Bases

We process data for the following purposes:

- Providing app functionality and backend communication.
- Operating push notifications.
- Security, abuse prevention, troubleshooting, and service reliability.
- Optional analytics and product improvement.
- Legal compliance and assertion/defense of legal claims.

Legal bases (where applicable under GDPR/EEA law) may include:

- Performance of a contract (Art. 6(1)(b) GDPR),
- Legitimate interests (Art. 6(1)(f) GDPR),
- Consent (Art. 6(1)(a) GDPR), especially for optional analytics where required,
- Legal obligation (Art. 6(1)(c) GDPR).

For analytics and product improvement in the current setup, processing is based on legitimate interests (Art. 6(1)(f) GDPR). Analytics tracking can be switched off in-app via settings. The analytics setting is enabled by default on first launch.
